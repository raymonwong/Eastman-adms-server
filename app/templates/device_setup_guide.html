Setup Steps设置步骤
Follow the screenshots in order. The exact local IP depends on the customer network.按截图顺序操作。本机 IP 需要根据现场网络确定。
1Open the Main Menu打开主菜单
  • On the home screen, tap the menu icon at the bottom right.在首页点击右下角的九宫格菜单图标。
  • If the device asks for permission, use an administrator or enrollment administrator account.如果设备要求权限，请使用管理员或登记管理员账号进入。
Home screen, menu icon at bottom right.首页，右下角进入菜单。
2Enter Communication Settings进入通讯设置
  • In the main menu, tap Communication Settings.在主菜单中点击 通讯设置。
  • The communication menu contains network settings and cloud server settings.通讯菜单中包含网络设置和云服务设置。
Main menu: choose Communication Settings.主菜单：选择通讯设置。
3Open Network Settings打开网络设置
  • Select Network Settings.选择 网络设置。
  • This section controls the local device IP, subnet mask, gateway, DNS, and TCP communication port.这里配置打卡机本机 IP、子网掩码、网关、DNS 和 TCP 通讯端口。
Communication menu.通讯设置菜单。
4Configure Local Device Network配置本机网络
  • Set a unique device IP in the local network, for example 192.168.137.200 or 192.168.1.201.设置一个现场局域网内不冲突的本机 IP，例如 192.168.137.200 或 192.168.1.201。
  • Subnet mask is usually 255.255.255.0.子网掩码通常为 255.255.255.0。
  • Gateway should be the router or shared-network gateway, for example 192.168.137.1.网关填写路由器或共享网络网关，例如 192.168.137.1。
  • Keep TCP communication port as 4370. Keep DHCP OFF for a stable device IP.TCP 通讯端口保持 4370。建议 DHCP 关闭，避免本机 IP 变化。
Example network configuration.网络配置示例。
5Configure ADMS Cloud Service配置 ADMS 云服务
  • Open Cloud Server Settings.进入 云服务设置。
  • Server Type: ADMS.服务器类型：ADMS。
  • Domain mode: OFF. Server Address: 47.110.79.69. Server Port: 4370.域名模式：OFF。服务器地址：47.110.79.69。服务器端口：4370。
  • Proxy: OFF. HTTPS: OFF.代理：OFF。HTTPS：OFF。
ADMS cloud service configuration.ADMS 云服务配置。
6Set Device Time and Verify Online设置设备时间并验证上线
  • Set the device date and time to Dubai time, UTC+4.将打卡机日期时间设置为迪拜时间 UTC+4。
  • Return to the home screen and confirm the displayed time is correct.返回首页，确认顶部显示的日期时间正确。
  • Open ADMS Operation Monitor. The device should become ONLINE and keep sending heartbeat requests.打开 ADMS 运行监控，设备应显示在线，并持续发送心跳请求。
  • After online, ADMS automatically sends employees, roles, and fingerprints.上线后，ADMS 会自动下发员工、角色和指纹。
Alternative network example. Confirm final values with the site network.另一组网络示例。最终参数按现场网络确认。

Final check最终检查
When the device is online, ADMS will show the device heartbeat, latest request, and synchronization progress. No SQL, SSH, or log analysis is required for normal verification. 设备上线后，ADMS 会显示设备心跳、最后请求和同步进度。正常验收不需要查询数据库、SSH 或分析日志。
