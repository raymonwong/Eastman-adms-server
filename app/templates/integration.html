1. System Overview / 系统概览
Mingdao is the master employee database. ADMS receives employee data from Mingdao through REST APIs, stores it in device_user, prepares per-device records in device_user_sync, and later communicates with attendance devices through ADMS GETREQUEST.

              
2. Authentication / 鉴权
Authorization: Bearer <API_TOKEN>
X-API-Key: <API_TOKEN>

              
3. Mingdao Configuration Guide / 明道配置说明
Configure Mingdao HTTP request nodes with the API URL shown on this page. Use method POST, header Content-Type: application/json, and either Authorization: Bearer <API_TOKEN> or X-API-Key: <API_TOKEN>.

              
4. Idempotency / 幂等规则
Repeated POST requests with unchanged payload do not create duplicate users or duplicate sync records.

              
5. Troubleshooting / 故障排查

Help and Troubleshooting / 帮助与故障排查

        

          

            

              1. Purpose / 用途
              
This page stores Mingdao OpenAPI configuration and can manually upload saved ADMS attendance events to Mingdao. The local sync table prevents the same ADMS attendance event from being uploaded twice.

            

            

              2. Mingdao OpenAPI Authentication / 明道 OpenAPI 鉴权
              
The connection test uses Mingdao OpenAPI headers copied from Mingdao.

              

                
Headers 

                
HAP-Appkey: <AppKey>
HAP-Sign: <Sign>

              

            

            

              3. Worksheet and Field Mapping / 工作表与字段映射
              
Configure the Mingdao Attendance Worksheet ID and target field IDs for Employee Record ID, Check Time, Device Name, and Device SN. The ADMS source fields are fixed; only Mingdao target field IDs are editable.

            

            

              4. Duplicate Prevention / 防重复上传
              
ADMS uses the local attendance event ID as the idempotency key. Synced records are never uploaded again. Records missing employee_record_id remain pending until employee data is completed.

            

            

              5. Troubleshooting / 故障排查
              

                

                  
IssueReasonSuggested Action

                  
                    
Connection test failedURL, AppKey, Sign, Worksheet ID, or Field ID is wrong.Check Mingdao OpenAPI settings and run the test again.

                    
Pending records remainDevice data exists but employee_record_id is missing.Sync Mingdao users first, then retry attendance sync.

                    
Failed records remainMingdao API or network returned an error.Check failure details and retry after the configured interval.
