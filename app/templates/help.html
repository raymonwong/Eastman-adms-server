System functions and user operation guide for daily ADMS administration. / ADMS 日常管理的系统功能说明与用户操作指南。
English first / 中文辅助
This page is a practical operating map. Use it to understand where each function lives, then follow the diagrams for employee creation, role setup, fingerprint enrollment, backup, and alerts.

System Functions / 系统功能
Operation Monitor

Shows server status, device online state, latest attendance, event log, and abnormal priorities.
显示服务器状态、设备在线、最近考勤、实时事件和异常优先级。

Device Management

Manages discovered devices, attendance recording, console visibility, and UTC+4 device notice.
管理自动发现设备、是否记录考勤、是否显示在监控页，以及 UTC+4 时区提示。

ADMS Users

Displays Mingdao users, PIN, employee record ID, sync state, fingerprint count, and device roles.
显示明道用户、打卡机 PIN、员工记录 ID、同步状态、指纹数量和设备角色。

Device Role Page

Assigns Normal, Device Admin, or Enroll Admin for each attendance device.
按设备给员工设置普通用户、管理员或登记管理员。

Inbound API

Receives employee data from Mingdao workflows and provides token, examples, and API test tools.
接收明道工作流员工数据，提供 Token、示例和接口测试。

Attendance Sync

Uploads ADMS attendance records to Mingdao with duplicate prevention, retry, and diagnostic log.
将 ADMS 考勤记录上传到明道，包含去重、重试和诊断日志。

Backup Settings

Supports daily backup, manual backup, retention, download, and restore instructions.
支持每日备份、手动备份、保留数量、下载和恢复说明。

Alert Settings

Monitors offline devices, sync failures, stale backup files, and optional webhook delivery.
监控设备离线、同步失败、备份过期，并可发送 Webhook 告警。

User Operation Guide / 用户操作说明
Add a New Employee / 新增员工
Mingdao Employee
→
POST /api/v1/users
→
ADMS User
→
Device GETREQUEST

When a Mingdao employee is created or updated, the workflow sends employee_record_id, employee_id, PIN, name, department, enabled, and privilege to ADMS.

明道新增或更新员工时，工作流把员工记录 ID、工号、PIN、姓名、部门、启用状态和权限发送到 ADMS。

  1. 1Create or update the employee record in Mingdao. 在明道创建或更新员工档案。
  2. 2The workflow calls the ADMS Inbound API. 工作流调用 ADMS 对外 API。
  3. 3ADMS queues synchronization for all attendance devices that record attendance. ADMS 为所有记录考勤的设备创建同步任务。
  4. 4The device receives the user on its next GETREQUEST. 设备下次 GETREQUEST 时接收用户。
Resignation or Disable / 离职或停用
Mingdao Status
→
enabled=false
→
ADMS Disabled
→
Device Disable

For resigned employees, send the same employee payload with enabled=false. ADMS marks the user as disabled and sends disable commands instead of deleting immediately.

员工离职时，发送同一员工数据并设置 enabled=false。ADMS 标记为停用，并下发停用命令，不立即删除。

Set Device Role / 设置打卡机角色
Select Device
→
Find Employee
→
Choose Role
→
Queued Command

Open the device role management link, select a device, then set the employee as Normal, Admin, or Enroll Admin. The role is applied on the next device request.

打开设备角色管理链接，选择设备，再把员工设置为普通用户、管理员或登记管理员。设备下次请求时应用角色。

Role / 角色Meaning / 含义
Normal User
普通用户		Punch attendance only. 仅用于打卡。
Device Admin
管理员		Full device administration permission according to device firmware. 按设备固件拥有完整管理权限。
Enroll Admin
登记管理员		ADMS assigns the enrollment role group; detailed permissions are configured on the device. ADMS 只分配登记角色组，细化权限在打卡机上配置。

Important Notes / 注意事项
Timezone / 时区

The current server is designed for UTC+4 attendance devices. For other timezones, contact development before adding devices.
当前服务器按 UTC+4 打卡机设计。其他时区新增设备前请联系开发升级。

Source of Truth / 数据源

Mingdao owns employee profile data. Attendance devices are terminals and should not overwrite employee names, departments, or enabled status.
员工基础资料以明道为准。打卡机只是终端，不覆盖姓名、部门和启用状态。

Backup Discipline / 备份习惯

Keep server backups and download important backup packages outside the server. This is the fastest recovery path after server failure.
服务器内保留备份，同时把重要备份包下载到服务器外，故障时恢复最快。
